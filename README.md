# AI Interviewer Platform

A lightweight voice-first interview practice app. Candidates select a sample job, answer role-grounded questions by microphone or typed fallback, and receive a saved transcript plus structured evaluation.

## Features

- Three sample jobs with title, description, competencies, and job-specific question packs.
- Interview room with browser speech recognition, typed fallback, and speech synthesis for interviewer questions.
- Deterministic interview engine that guarantees six questions and two answer-dependent follow-ups.
- Decision panel showing rubric signals, skills detected, topics covered, gaps, and next-question rationale.
- Optional camera preview for video-call-style UX; interview remains voice-driven.
- Local session persistence with replay, role filtering, duration, talk ratio, topic coverage, and score trend analytics.
- Server-side API route that can enhance questions/evaluations with an LLM when `OPENAI_API_KEY` is configured.
- Deterministic fallback works end-to-end without secrets.
- GitHub Actions workflow for building a macOS desktop artifact on a macOS runner.

## Tech stack

- Next.js + React + TypeScript
- Browser Web Speech API for microphone transcription where supported
- Browser speech synthesis for interviewer playback where supported
- localStorage for session persistence
- Vitest for deterministic engine and persistence tests
- Electron + electron-builder for Mac artifact generation

## Optional LLM enhancement

The app works without an API key. To enable server-side LLM enhancement for question wording and final evaluation, copy `.env.example` to `.env.local` and set:

```bash
OPENAI_API_KEY=your_key_here
OPENAI_MODEL=gpt-4o-mini
```

The key is only read by the server-side API route. It is not exposed to the client.

## Deploy to a public website

The app is deployed on Vercel at https://afterquery-psi.vercel.app.

To deploy another instance on Vercel, import the public GitHub mirror or this repository and use the default Next.js settings:

- Install command: `npm ci`
- Build command: `npm run build`
- Output: Next.js default

Add `OPENAI_API_KEY` and `OPENAI_MODEL` only if LLM enhancement is desired. The verified deployment currently runs the deterministic fallback path.

For a generic Node host:

```bash
npm ci
npm run build
npm run start
```

Serve the process behind HTTPS so browser microphone permissions work reliably.

## Mac downloadable artifact

Direct macOS packaging is not available from this Linux environment. The repository includes `.github/workflows/mac-build.yml`, which builds the Mac artifact on GitHub's macOS runner.

To produce the Mac download after pushing to GitHub:

1. Open the repository on GitHub.
2. Go to **Actions**.
3. Select **Build macOS Desktop Artifact**.
4. Click **Run workflow**.
5. When the workflow completes, download the `ai-interviewer-platform-macos` artifact from the run summary.

The artifact should contain `.dmg` and `.zip` outputs generated by `electron-builder`.

## Notes and limitations

- Speech recognition support varies by browser. The typed fallback is always available.
- Sessions are stored locally in the current browser and are not shared across devices.
- The offline Mac wrapper uses the deterministic fallback path; server-side LLM routes are only available in the hosted Next.js deployment.
- Camera input is UX-only and is not recorded or analyzed.
